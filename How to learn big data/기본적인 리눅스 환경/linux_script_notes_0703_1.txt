리눅스란 ?
- 핀란드 태생의 리누스 토발즈(Linux B.Torbalds) 1991년 9월 17일 0.01버전 개발
- 앤드류 타넨바움(Andrew S.Tanenbaum)이 학습을 주목적으로 만들었던 미닉스(Minix)를 사용
- 1991년 10월 5일 리눅스 커널 첫 버전 공식버전인 0.02를 뉴스 그룹에 발표 
리눅스 특징 
- 실시간 페이지 적재 기능 
- CPU 최적화
- 뛰어난 네트워킹
- 가상콘솔
- POSIX의 완벽한 호환
- 트리(Tree)형태의 계층적 구조의 파일 시스템
- 효율적인 가상 메모리
- 공적 공유 라이브러리 제공
- 완벽한 다중 사용자, 다중 작업 시스템
- 다중 플랫폼
- 최신 웹서비스 제공

리눅스 장단점?
- 이식성과 확장성이 용이
- 텍스트 모드 중심의 관리와 다양한 관리 환경의 제공
- 풍부한 소프트웨어 개발 환경을 제공
- 다양한 네트워크 서비스 및 작업환경 제공
- 뛰어난 안정성
- 시스템 보안성
- 폭넓으 하드웨어 장치 지원
- 저수준 하드웨어 구성된 시스템의 사용 가능
- 시스템의 높은 신뢰성
- 가격 대비 탁월한 성능

root의 개념 
- 리눅스 시스템으 운영하기 위한 모든 권한을 가지며 수퍼 유저라고 함

디렉토리 구조
- /bin 기본적인 명령어가 저장된 디렉토리. mv, cp, rm, rmdir, df, sync등과 같은 명령어들이 이 디젝토리에 존재
  /usr/bin CentOS 7부터는 /usr/bin으로 심볼릭 링크됨.

- /boot 리눅스 부트로더(Boot Loader)가 존재하는 디렉토리. GRUB와 같은 부트로더에 관한 파일들이 이 디렉토리에 존재함.
- /dev 시스템 장치(device)파일을 저장하고 있는 디렉토리.// like cdrom, print etc.... device in there
- /etc 시스템의 거의 모든 설정 파일이 존재하는 디렉토리.
- /lib 커널 모듈 파일과 라이브러리 파일들이 존재함. C, C++등등
- /media DVD, CD-ROM, USB메모리 스틱과 같은 탈부탁이 가능한 장치들의 마운트 포인트(mount point).
- /opt 추가적인 소프트웨어를 설치하는 디렉토리.
- /sbin 시스템 관리를 위해 사용되는 유틸리티들. 
- /svr 시스템에 의해 제공되는 서비스들에 대한 자료.
- /tmp 임시 파일들이 저장ro
- /usr 2차적 디렉토리 체계. 시스템이 아닌 일반 사용자들이 주로 사용하는 디렉토리.
- /var 디렉토리는 시스템 운용 중에 생성되었다가 삭제되는 자료.
- /home 사용자의 홈디렉토리 위치.
- /root root사용자를 위한 홈 디렉토리.
- /proc 커널과 프로세스 정보를 위한 가상 파일 시스템.
